# 📱 제스처 네비게이션 가이드

## 좌우 스와이프로 페이지 이동

앱의 모든 화면에서 좌우로 스와이프하여 이전/다음 페이지로 쉽게 이동할 수 있습니다.

### 🔄 제스처 사용법

#### **← 오른쪽으로 스와이프 (이전 페이지)**
- 화면을 왼쪽에서 오른쪽으로 드래그
- 이전 화면으로 돌아갑니다
- 예: 팀 구성 → 출석 체크

#### **→ 왼쪽으로 스와이프 (다음 페이지)**
- 화면을 오른쪽에서 왼쪽으로 드래그
- 다음 화면으로 이동합니다
- 예: 출석 체크 → 팀 구성

### 📋 페이지 흐름

```
홈 → 경기 선택
  ↓
출석 체크 (← → 팀 구성)
  ↓
팀 구성 (← → 경기 결과)
  ↓
경기 결과 (←)
```

### 🎯 활용 예시

**1. 빠른 뒤로 가기**
- 버튼을 찾지 않고 화면을 오른쪽으로 스와이프
- 바로 이전 화면으로 이동

**2. 순차적 진행**
- 출석 체크 완료 후 왼쪽으로 스와이프
- 자동으로 팀 구성 화면으로 이동

**3. 빠른 네비게이션**
- 여러 화면을 연속으로 스와이프하여 빠르게 이동

### ⚙️ 제스처 설정

**민감도:**
- 최소 스와이프 거리: 50px
- 최대 스와이프 시간: 500ms
- 수직 이동 제한: 100px (대각선 스와이프 방지)

**적용 화면:**
- 모든 Stack Navigator 화면
- 전체 화면 제스처 활성화
- 드래그 앤 드롭과 충돌하지 않음

### 📱 모바일 앱 (React Native)

**iOS:**
- 기본 제스처 자동 활성화
- 화면 가장자리부터 스와이프 시작

**Android:**
- 전체 화면에서 제스처 작동
- 부드러운 애니메이션 효과

### 🌐 HTML 목업 (웹 브라우저)

**테스트 방법:**
1. 모바일 브라우저 또는 개발자 도구 (F12)에서 모바일 모드 활성화
2. 터치/드래그로 좌우 스와이프
3. 페이지 전환 확인

**지원 브라우저:**
- Chrome (권장)
- Safari
- Edge
- Firefox

### ⚠️ 주의사항

1. **드래그 앤 드롭과 구분**
   - 회원 동그라미를 드래그할 때는 제스처가 작동하지 않습니다
   - 빈 공간이나 헤더 영역에서 스와이프하세요

2. **스크롤과 구분**
   - 수직 스크롤 시에는 제스처가 작동하지 않습니다
   - 수평 방향으로만 드래그해야 합니다

3. **첫 화면과 마지막 화면**
   - 첫 화면에서 오른쪽 스와이프: 작동 안 함
   - 마지막 화면에서 왼쪽 스와이프: 작동 안 함

### 💡 팁

- **빠른 스와이프가 더 효과적입니다**
  - 천천히 드래그하면 인식되지 않을 수 있습니다
  
- **화면 가장자리에서 시작**
  - 화면 왼쪽 또는 오른쪽 가장자리에서 시작하면 더 잘 인식됩니다
  
- **애니메이션 확인**
  - 스와이프 시 "이전/다음 페이지로 이동 중..." 메시지가 표시됩니다
